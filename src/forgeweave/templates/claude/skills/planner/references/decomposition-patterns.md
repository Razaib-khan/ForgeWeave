# Common Task Decomposition Patterns

## Pattern 1: Feature Implementation
1. Design architecture
2. Scaffold module structure
3. Implement data layer
4. Implement business logic
5. Implement UI/components
6. Write tests
7. Document

## Pattern 2: Bug Fix
1. Reproduce the bug
2. Isolate root cause
3. Write failing test
4. Implement fix
5. Verify fix passes test
6. Check for regressions

## Pattern 3: Performance Optimization
1. Profile to identify bottleneck
2. Hypothesis: what's causing it
3. Implement targeted fix
4. Profile again to confirm improvement
5. Repeat if needed

## Pattern 4: Refactoring
1. Ensure existing tests pass
2. Identify code smells
3. Apply one refactoring at a time
4. Run tests after each change
5. Repeat

## Pattern 5: Research
1. Decompose topic into subtopics
2. Gather sources per subtopic
3. Extract findings
4. Cross-validate
5. Synthesize report
